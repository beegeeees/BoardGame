## 1. 멀티플레이 동기화 및 플레이어 슬롯

### 플레이어 슬롯 고정

- 서버의 `Player`와 소켓의 `PlayerSnapshot`에 `slotIndex`를 추가했다.
- 플레이어가 방에 입장할 때 비어 있는 슬롯 번호를 서버가 배정한다.
- 중간 플레이어가 나가더라도 남은 플레이어의 P 번호와 색상이 밀리지 않는다.
- 대기방과 보드판은 플레이어 목록 순서가 아닌 `slotIndex`를 기준으로 표시한다.
- 게임 턴 순서도 슬롯 번호를 기준으로 생성한다.

### 대기방 표시 개선

- 방 코드 옆에 현재 플레이어의 P 번호를 표시한다.
- 현재 사용자를 `[나]`로 구분한다.
- 방장, 준비 완료, 준비 중 상태 표시를 정리했다.
- 비밀번호 방 입장 화면의 힌트를 `비밀번호 입력`으로 변경했다.

## 2. 보드판 및 주사위 흐름

### 보드 내부 주사위 UI

- 별도의 `DiceActivity`로 이동하던 흐름을 보드 중앙 오버레이 방식으로 변경했다.
- 기존 주사위 선택 확률, 굴리기 애니메이션, 다크 패턴을 유지했다.
- 다크 패턴은 버튼의 실제 위치를 교환하는 방식으로 개선했다.
- 주사위 화면에서 `보드 보기`를 선택할 수 있다.
- 보드로 돌아간 뒤 주사위 버튼을 누르면 진행 중이던 화면을 다시 연다.

### 주사위 제한 시간

- 제한 시간을 10초에서 30초로 변경했다.
- 주사위 버튼을 누른 시점이 아니라 내 턴이 시작되는 즉시 카운트한다.
- 보드 화면에서도 남은 시간을 확인할 수 있다.
- 시간이 끝나면 `0`을 표시한 뒤 자동으로 주사위를 굴린다.
- 서버 제출 대기 중 타이머가 중복 시작되지 않도록 상태를 분리했다.

### 보드 표시 개선

- 점수판의 `[나]` 문자열을 실제 둥근 `나` 배지로 변경했다.
- 현재 턴 점수판을 확대하고 강조한다.
- 플레이어 말과 점수판을 고정 슬롯 기준으로 표시한다.
- 동점 플레이어는 공동 순위로 처리한다.
- 최종 라운드가 `4/3`으로 표시되던 문제를 수정했다.

## 3. 보드 칸 이벤트 안내

### 카드 칸

- 방어 카드 획득 결과를 팝업으로 표시한다.
- 이미 카드를 가지고 있다면 중복 획득이 되지 않았음을 안내한다.
- 방어 카드가 감점 칸에서 자동으로 사용된다는 설명을 제공한다.

### 물음표 칸

- `+5점` 또는 `-5점` 결과를 팝업으로 표시한다.
- 이벤트를 발생시킨 플레이어에게만 결과 팝업을 표시한다.

### 광고 및 라운드 미니게임 전환

- 광고 칸 도착 후 약 3초간 안내 팝업을 표시한 뒤 광고 게임을 시작한다.
- 마지막 플레이어 턴 종료 후 약 5초간 안내 팝업을 표시한 뒤 라운드 미니게임을 시작한다.
- 광고 게임 성공 점수는 `+5`, 실패 점수는 `-5`로 통일했다.

## 4. 라운드 미니게임

### 공통 준비 화면

- 준비 화면을 흰색 배경과 검은색 계열 글자로 통일했다.
- 비밀번호 미니게임을 세로 화면으로 고정했다.
- 불필요한 화면 회전 안내 문구를 제거했다.
- 설명 화면과 시작 카운트다운 흐름을 통일했다.

### 공통 클리어 피드백

- 클리어 순간 화면 중앙에 큰 체크 표시와 완료 문구를 약 2초간 표시한다.
- 캡챠, 비밀번호, 볼륨 미로에 동일한 완료 피드백을 적용했다.
- 기존 점수 제출 및 전체 미니게임 종료 대기 시간은 유지한다.

### 캡챠

- 게임 화면 배경을 흰색으로 변경했다.
- 60초 제한 시간을 추가했다.
- 제한 시간 종료 시 성공 여부와 진행도를 반환한다.
- 5단계 완료 후 입력을 막고 클리어 상태를 명확히 표시한다.

### 비밀번호

- 각 스테이지 준비 및 결과 화면을 세로 형식으로 정리했다.
- 결과 화면 점수 안내를 실제 서버 규칙인 `10/7/5/3`으로 변경했다.
- 미완료 시 진행도 0으로 제출된다는 내용을 표시한다.
- 첫 스테이지 초기화 중 조건 수가 0이라 자동 클리어되던 버그를 수정했다.

### 볼륨 미로

- 게임을 열 때마다 미로 구조를 랜덤 생성한다.
- 시작점에서 도착점까지 경로가 존재하는지 BFS로 자동 검증한다.
- 검증에 실패한 미로는 최대 8번 다시 생성한다.
- 계속 실패하면 기존 고정 시드 `20260427` 미로를 사용한다.
- 기본 경로에 서로 겹치지 않는 우회 통로를 1~2개 추가해 2~3개의 선택지를 제공한다.
- 초기화 시 공은 가장 왼쪽 아래로 돌아간다.
- 초기화해도 파란 자석 위치와 현재 미로 구조는 유지한다.

## 5. 점수 및 게임 규칙

- 라운드 미니게임 순위 보상을 `30/20/10/5`에서 `10/7/5/3`으로 변경했다.
- 동점 플레이어는 동일한 순위 보상을 받는다.
- 최종 점수가 같은 플레이어를 공동 우승자로 표시한다.
- 광고 게임 서버 제출의 최솟값을 `-5`로 확장했다.
- 카드 목록은 실제 구현된 방어 카드만 남겼다.
- 물음표 이벤트 설명을 실제 구현인 `+5` 또는 `-5`로 정리했다.
- 규칙 문서와 앱 내 How to play 내용을 실제 동작에 맞췄다.

## 6. 옵션 화면

- 동작하지 않거나 제출 범위에서 제외된 옵션을 제거했다.
  - 볼륨 조절
  - 쉬운 볼륨 모드
  - 사운드 효과
  - 진동
  - 주사위 속도
  - 디버그 모드
- 서버 주소 설정 기능은 유지했다.
- 옵션 화면에서도 현재 닉네임을 확인하고 변경할 수 있도록 했다.
- 제거된 옵션에 사용되던 `SessionPrefs` 설정 코드도 정리했다.

## 7. 빌드 환경

- 프로젝트 공용 `gradle.properties`에서 Linux 전용 절대 경로인
  `org.gradle.java.home=/usr/lib/jvm/java-21-openjdk-amd64`를 제거했다.
- 팀원별 JDK 경로 차이로 Gradle 실행이 실패하는 문제를 방지한다.
- 각 개발자는 Android Studio의 Gradle JDK 설정 또는 로컬 환경에서 JDK를 지정한다.

## 9. 제출 전 확인 사항

1. 새 Java 파일 2개와 이 문서를 Git에 추가한다.
2. Android Studio에서 앱 빌드 또는 APK 생성을 실행한다.
3. 2명 이상 접속 후 턴 전환과 30초 주사위 타이머를 확인한다.
4. 카드, 물음표, 광고 칸의 팝업과 점수 반영을 확인한다.
5. 세 라운드 미니게임의 클리어 표시와 점수 제출을 확인한다.
6. 볼륨 미로를 여러 번 실행해 경로 생성과 초기화 동작을 확인한다.
7. 최종 확인 후 커밋하고 `master`를 push한다.
