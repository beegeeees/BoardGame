# 1. 기본 구조

불편마불은 4인 멀티플레이 턴제 보드게임이다.

게임은 크게 두 단계가 반복된다.

보드판 페이즈

미니게임 페이즈

플레이어 4명이 각각 한 번씩 주사위를 굴린 후 4인 동시 미니게임이 진행된다.
(플레이어 1 주사위 굴림 -> 플레이어 2 주사위 굴림 -> 플레이어 3 주사위 굴림 -> 플레이어 4 주사위 굴림 -> 4인 미니게임)

이 과정을 총 3라운드 반복하고, 최종 누적 점수가 가장 높은 플레이어가 승리한다.

# 3. 보드판 구조

보드판 크기

보드판은 총 16칸으로 구성된다.

플레이어는 시작 칸에서 출발하며, 주사위를 굴려 나온 숫자만큼 이동한다.

이동 방식

보드판은 순환 구조이다.

15번 칸 다음은 다시 0번 시작 칸으로 이어진다.

한 바퀴를 돌아도 별도 보너스는 기본적으로 없다.

필요하면 추후 “한 바퀴 완주 보너스”를 추가할 수 있다.



# 4. 턴 진행 방식

다음 플레이어 턴으로 전환



# 5. 주사위 굴리기 규칙

기본 규칙

주사위는 1~6 사이의 숫자가 나온다.

주사위 컨트롤: (1~2 버튼) (3~4 버튼) (5~6 버튼) (랜덤 버튼)이 각각 있어 주사위 눈을 어느 정도 원하는 숫자로 보정할 수 있는 시스템(특정 눈이 나올 확률 증가)

(1~2 버튼) 30% 30% 10% 10% 10% 10%

(3~4 버튼) 15% 15% 20% 20% 15% 15%

(5~6 버튼) 15% 15% 15% 15% 20% 20%

(랜덤 버튼) 전부 1/6



간헐적으로 플레이어에게 다크 패턴 UI 기믹이 발동하여 단순히 버튼을 누르는 것이 아니라, 기믹을 통과해야 주사위를 굴릴 수 있다.

주사위 기믹은 확률적으로 등장한다. (20%)

주사위 기믹 예시 (‘이용약관에 동의하고 싶어’ 게임을 참고하면 좋을 듯)

버튼들이 갑자기 가려진 뒤 여러 버튼끼리 섞임(야바위)

(1~2 버튼) (3~4 버튼) (5~6 버튼) (랜덤 버튼)

판정 기준

가려진 뒤 섞인 버튼 중 임의의 버튼을 누르면 가려졌던 글자가 공개되며 그 버튼의 효과가 적용됨

예) 플레이어 A는 5~6 버튼을 누르려 했으나 야바위 기믹이 발동하여 버튼들의 글자가 가려지고 위치가 섞임. 플레이어 A는 다시 원래 5~6 버튼이 있던 위치의 버튼을 눌렀으나 그 위치에는 1~2 버튼이 있었고 주사위는 1, 2 확률이 높아진 채로 굴려지게 됨.



# 6. 점수 규칙

기본 점수

플레이어는 주사위로 이동한 칸 수에 따라 기본 점수를 얻는다.

예:

주사위 1 → +1점

주사위 2 → +2점

주사위 3 → +3점

주사위 4 → +4점

주사위 5 → +5점

주사위 6 → +6점

즉, 이동 거리 자체가 기본 점수가 된다.

칸 효과 점수

도착한 칸의 종류에 따라 추가 점수 변화가 발생한다.

최종 점수 변화는 다음과 같다.

턴 점수 변화 = 주사위 기본 점수 + 도착 칸 효과 점수



# 7. 타일 종류

보드판에는 다음 종류의 칸이 존재한다.

## ① 시작 칸

모든 플레이어의 출발 지점이다.

기본적으로 특별한 효과는 없다.

선택 규칙:

시작 칸에 도착하면 +5점

시작 칸을 지나가도 보너스 없음



## ② 점수 증가 칸

도착 즉시 점수를 얻는다.

예:

+3점

+5점

+10점

추천:

점수 증가 칸은 +3점으로 통일한다.

너무 다양한 점수는 밸런스가 어려워진다.



## ③ 점수 감소 칸

도착 즉시 점수를 잃는다.

예:

-3점

-5점

-10점

추천:

점수 감소 칸은 -3점으로 통일한다.

단, 점수는 0점 아래로 내려갈 수 있게 할지 정해야 한다.

추천 규칙:

점수는 음수가 될 수 있다.

이유: 불편마불 컨셉상 패널티가 명확한 편이 좋다.



## ④ 물음표 칸

무작위 이벤트가 발생하는 칸이다.

이 칸은 게임의 변수를 만드는 역할을 한다.

물음표 이벤트 후보

1등과 점수 교환

꼴등과 점수 교환

랜덤 플레이어와 위치 교환

다음 주사위 값 2배

다음 주사위 값 절반(소수점은 올림)

점수 +7

점수 -7

모든 플레이어 점수 2배

모든 플레이어 점수 절반(소수점은 올림)

자신을 제외한 모든 플레이어 점수 -3



## ⑤ 카드 칸

도착 시 소모성 카드를 1장 획득한다.

카드는 바로 발동되지 않고, 보관했다가 특정 타이밍에 사용할 수 있다.

카드 보유 제한

추천 규칙:

플레이어는 최대 1장의 카드만 보유할 수 있다.

이미 1장을 가진 상태에서 카드 칸에 도착하면,
새 카드 1장을 뽑고 기존 카드 중 1장을 버릴 수 있다.



# 8. 카드 종류

## ① 방어권 카드

다음에 받는 점수 감소 효과를 1회 무효화한다.

적용 대상:

점수 감소 칸

물음표 부정 이벤트

광고 칸 실패 패널티

적용 제외:

미니게임 순위 결과

주사위 기본 점수



## ② 10 주사위 카드

이번 턴의 주사위 값이 1~10이 된다.



## ③ 456 주사위 카드

이번 턴의 주사위 값이 4~6이 된다.



## ④ 주사위 홀짝 카드

이번 턴의 주사위 값을 홀수 또는 짝수만 나오게 설정한다.

사용 방식:

사용할 시 홀수와 짝수 버튼이 나오며 선택한 결과에 따라 주사위 값이 정해진다.



## ⑤ 광고 스킵 카드

광고 칸에 도착했을 때 광고 미니게임을 건너뛴다.

효과:

광고 미니게임을 진행하지 않음

사용 시 +3점

# 11. 미니게임 페이즈

4명의 플레이어가 모두 한 번씩 턴을 진행하면 미니게임 페이즈로 전환된다.

미니게임 시작 조건

모든 플레이어가 이번 라운드의 보드 턴을 완료

서버가 미니게임 종류를 결정

모든 클라이언트에 미니게임 시작 신호 전송

미니게임 종류

색상 캡챠

비밀번호 생성

볼륨 미로

미니게임 선택 방식

추천 규칙:

1라운드: 색상 캡챠
2라운드: 비밀번호 생성
3라운드: 볼륨 미로

이유: 발표와 구현에서 설명하기 쉽고, 랜덤 선택보다 테스트가 쉽다.

선택 규칙으로 바꾸고 싶다면:

매 라운드마다 중복 없이 랜덤 선택



# 12. 미니게임 점수 규칙

미니게임은 클리어 순위에 따라 점수를 지급한다.

추천 점수:

타임오버 처리

제한 시간 안에 아무도 클리어하지 못할 수 있다.

이 경우 두 가지 방식 중 하나를 선택해야 한다.

추천 방식:

타임오버 시 진행도 기준으로 순위를 계산한다.

예:

캡챠: 도달한 단계가 높은 순

비밀번호: 해금한 조건 개수가 많은 순

볼륨 미로: 볼륨 진행도가 높은 순

진행도가 같으면 해당 진행도에 먼저 도달한 플레이어가 우선이다.

더 단순한 방식:

타임오버 시 모두 0점

# 16. 서버로 보내는 데이터

보드판 페이즈

클라이언트가 서버로 보내는 정보:

주사위 액션 완료

주사위 결과

도착 칸

카드 사용 요청

광고 미니게임 성공/실패

단, 최종 점수 반영은 서버가 담당하는 것이 안전하다.

미니게임 1: 캡챠

상태가 변할 때만 전송한다.

전송 예:

stage: 2, state: PROGRESS

stage: 1, state: RESET

stage: 5, state: CLEAR

미니게임 2: 비밀번호

문자열 자체는 보내지 않는다.

전송 예:

unlockedCount: 4, state: PROGRESS

unlockedCount: 10, state: CLEAR

미니게임 3: 볼륨 미로

센서값이나 충돌 데이터는 보내지 않는다.

서버팀 설명과 맞추려면 최종 결과 중심으로 정리한다.

전송 예:

state: CLEAR

completionTime

score

진행도 기반 타임오버 판정을 쓰려면 다음 중 하나를 선택해야 한다.

선택 A: 진행도도 구간별 전송

10%, 20%, 30% 단위로 전송

타임오버 시 진행도 순위 계산 가능

선택 B: 클리어 결과만 전송

서버 부담 최소화

단, 타임오버 시 진행도 순위 계산 불가능

타임오버는 0점 처리로 해야 함

서버팀과 맞추려면 추천은 B다.

볼륨 미로는 클리어 결과만 서버로 전송하고, 타임오버 시 미클리어 플레이어는 0점 처리한다.



# 17. 공정성 규칙

왜 클라이언트 판정을 쓰는가?

실시간 반응성이 중요하기 때문이다.

서버가 모든 입력을 검사하면:

지연 발생

센서 처리 어려움

플레이 감각 저하

네트워크 부하 증가

그래서 클라이언트가 실시간 처리를 하고,
서버는 결과와 상태만 관리한다.

치팅 문제는?

클라이언트 판정은 조작 가능성이 있다.

보완 방법:

클리어 시간이 비정상적으로 빠르면 무효 처리

진행도 변화가 비정상적이면 무효 처리

서버 기준 타임스탬프 사용

미니게임 시작 전 서버가 seed 제공

결과 제출은 해당 미니게임 상태에서만 허용
